PHASE 2 - API LAYER (src/api/)
============================================================
[ ] 2.1 src/api/http.js
      - axios.create({ baseURL: API_BASE_URL, withCredentials: true })
      - response interceptor: on 401, emit a global "unauthenticated" event
        (e.g. via a tiny event bus or router redirect to /admin/login)
      - response interceptor: normalize error -> throw Error(response.data) so
        callers get the plain-text backend message
[ ] 2.2 src/api/memberApi.js
      - getMemberInfo(secretKey): GET /member/{secretKey}
      - checkIn(secretKey): POST /member/checkin/{secretKey}
[ ] 2.3 src/api/sessionApi.js
      - getAllSessions(): GET /admin/sessions
      - getSessionMembers(sessionId): GET /admin/sessions/members/{id}
      - finalizeSession({sessionId, sessionType}): POST /admin/finalizeSession
[ ] 2.4 src/api/adminMemberApi.js
      - getAllMembers(): GET /admin/members
      - createMember({name, regularTickets, commitTickets}): POST /admin/member
      - addTickets({memberId, commitTickets, regularTickets}): POST /admin/tickets
[ ] 2.5 src/api/authApi.js
      - login(username, password): POST /api/login as
        application/x-www-form-urlencoded with fields "username"/"password"
        (Spring formLogin default) -- axios needs URLSearchParams, not JSON
      - logout(): POST /api/logout
[ ] 2.6 Write a tiny manual test (or curl) for each call against a running
      backend before wiring up UI, to confirm request shape matches Spring's
      expectations (esp. login's form-encoding and CORS credentials)

============================================================
[ ] 7.1 Integrate html5-qrcode: start camera scanner on mount, stop on unmount
