CHOIR VUE FRONTEND - TODO v1 (OVERVIEW / MILESTONES)
======================================================
Backend reference: choir-backend already implements Member/Session/Attendance
endpoints under /member and /admin (Spring Boot, HTTP Basic + session-cookie
auth on /admin/**). This checklist is about building the Vue frontend against
that existing API.

[ ] 1. Project setup
      - Add router, HTTP client, QR generation + QR scanning libs
      - Configure API base URL via .env

[ ] 2. API layer
      - One module per backend resource (member, admin/sessions, admin/members)
      - Central place for auth (credentials, 401 handling)

[ ] 3. App shell & routing
      - Routes for Member view, Admin view, Doorman view, Admin login
      - Basic layout/nav per role

[ ] 4. Member Dashboard (MVP)
      - Ticket count, last attended sessions, login status
      - Special link/QR code display
      - Check-in action

[ ] 5. Admin Dashboard - Sessions (MVP)
      - Session table with member counts
      - Close session as REGULAR_ONLY or COMMIT
      - View attended members per session

[ ] 6. Admin Dashboard - Members (MVP)
      - Member table (name, tickets, secret key/QR export)
      - Create member, add tickets

[ ] 7. Doorman Dashboard (MVP)
      - QR scanner -> check in member
      - Green/red result screen with remaining tickets

[ ] 8. Admin authentication
      - Login form using backend session-based auth
      - Guard admin routes, handle 401/logout

[ ] 9. Stretch features (marked * in general-idea.md)
      - PayPal link, mailto links to app/choir admin
      - Session data export
      - Messages into member dashboard (needs backend support)

[ ] 10. Cross-cutting polish
      - Error/loading states, mobile responsiveness (doorman uses phones)
      - Env-based config for deployment (IONOS/Docker)
