CHOIR VUE FRONTEND - TODO v2 (DETAILED)
======================================================
Backend API surface confirmed by reading choir-backend source:

  Public:
    GET  /member/{secretKey}            -> {name, regularTickets, commitTickets, pastAttendances:[{attendedOn,status}]}
    POST /member/checkin/{secretKey}    -> 200 empty body | 404 not found | 400 already checked in / insufficient tickets

  Admin (requires authenticated session, HTTP Basic or /api/login form):
    POST /admin/finalizeSession {sessionId, sessionType: REGULAR_ONLY|COMMIT} -> {presentMembers, absentCommitMembers}
    POST /admin/member {name, regularTickets, commitTickets}                 -> {secretKey}
    GET  /admin/members                                                     -> [{id,name,regularTickets,commitTickets,secretKey}]
    POST /admin/tickets {memberId, commitTickets, regularTickets}           -> void (values are DELTAS, added to existing)
    GET  /admin/sessions                                                    -> [{id,startTime,isOpen,sessionType,amountOfAttendees}]
    GET  /admin/sessions/members/{id}                                       -> [{id,name,regularTickets,commitTickets,secretKey}]

  Auth endpoints: POST /api/login (username/password form fields), POST /api/logout
  Error bodies are plain text strings with matching HTTP status (400/404/500).
